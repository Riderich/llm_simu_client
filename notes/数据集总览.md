## 各数据集可用性评估

### ESConv ✅ 主力数据集
- **优势**：格式规范，结构化好，已被大量工作使用，易于处理
- **问题**：原始数据为众包模拟对话，来访者容易被说服，阻抗表现较弱；需要自行识别并标注阻抗句
- **用于**：阻抗自动标注（大模型 → 人工 review）→ SFT 训练数据

### ExTES ⚠️ 阻抗稀少，不作主力，可作 no-resistance 补充语料

> **质量分析**：2026-03-10，随机抽取 40 条对话（seed=42），使用 DeepSeek-V3 对所有 User utterance 逐句进行 PsyFIRE 阻抗分类（原始结果保存于 `extes_resistance_analysis.json`）

#### 📊 实测统计（LLM 分析）

| 指标 | 数值 |
|------|------|
| 分析 User utterances 总数 | 351 句 |
| 识别为阻抗的 utterances | **21 句（6.0%）** |
| 含有阻抗 utterances 的对话 | **17/40（42.5%）** |
| 每条有阻抗对话的平均阻抗句数 | ~1.2 句 |

#### 🗂️ 阻抗类别分布（LLM 标注）

| 类别 | 出现次数 |
|------|---------|
| Denying 各子类（Pessimism / Excusing / Reluctance）合计 | **16** |
| Reluctance（独立标注） | 2 |
| Avoiding | 1 |
| Unwillingness | 1 |
| Pessimism（独立标注） | 1 |

#### 🔍 质性观察

1. **对话为「线性配合型」**：典型模式为 User 倾诉 → AI 给建议 → User 接受 → 感谢结束。来访者极少持续 push back，多数对话在 2-3 轮后进入正向配合
2. **阻抗表现极为温和**：即便 LLM 标注为 Denying，实际上也多是短暂悲观情绪（如"我不确定这有没有用"），随后立即软化配合，不构成临床意义上的持续性阻抗
3. **对话弧线短促**：平均 15.1 轮，缺少真实多轮拉锯的张力
4. **字段格式轻微不统一**：约 0.02% 的条目存在字段名变体（`User A`、`AI_strategy`、`AI Strategies` 等），解析时需容错处理；`scene` 字段有 2 条缺失
5. **AI Strategy 标注质量好**：覆盖率约 99%，16 种策略分类清晰，**可用于研究 supporter 策略体系**

#### ✅ 结论

- **不推荐**作为阻抗识别标注或阻抗 SFT 训练的主力数据（阻抗比例仅 6%，且质量偏弱属浅层表达）
- **可参考**用于：① 研究 support 策略标注体系；② 作为"无阻抗（no resistance）"补充语料，防止模型产生 bias
- **local path**：`workspace/dataset/ExTES.json`（44.3MB）、`workspace/dataset/ExTES_strategies.json`

### MESC ✅ 高质量补充
- **优势**：真实心理治疗场景（非众包模拟），来访者为真实患者，阻抗表现更真实；对话轮次充足（28.2 轮）
- **问题**：多模态数据，文本提取可能需要额外处理；场景为 TV 剧，存在戏剧化成分
- **用于**：补充 ESConv 的阻抗标注数据，提升训练数据多样性

### AnnoMI ✅ 阻抗标注金标准（英文）
- **优势**：已有专家级别的 `client_talk_type` 标注（sustain/change/neutral，对应阻抗/配合/中性）；已处理为 JSON 格式，已有 PsyFIRE 细化标注（32 条 transcript）
- **问题**：总量相对较小（133 transcripts），且 sustain 比例仅 11.2%（541 句），纯阻抗数据量有限
- **当前状态**：已完成 Inner Monologue 批量生成（全量 153 条，stride=10），是目前最可用的训练数据
- **local path**：`data/raw/AnnoMI/`，`data/processed/AnnoMI-full.json`

### RECAP (ClientResistance) ✅ 中文阻抗标注金标准（新入库 2026-03-17）
- **优势**：
  - 5,154 条真实中文心理咨询标注数据，是 AnnoMI 阻抗句数的近 8 倍
  - 14 类细粒度标签体系与我们的行为空间框架（A/B/C/D/E）直接对齐（中文命名略有差异，映射见下）
  - 每条数据含完整多轮上下文，最后一句来访者话语为被标注的目标句，结构完全可用
  - 全量阻抗标签已覆盖（不需要再跑额外 LLM 标注），可直接作为 SFT 训练数据来源
- **问题**：
  - 纯中文，与现有英文数据集（AnnoMI/ESConv/MESC）存在语言差异，需考虑分语言训练或混合策略
  - RECAP 模型权重暂未到位（协议待签），暂时只能用其标注数据，不能用其推理能力对 ESConv/MESC 打标
  - 标注粒度为「末尾 client 句 + 上下文」打包式，非 utterance 级别逐句标注（但实际可等价使用）
- **标签映射（RECAP 中文 → 我们的框架）**：

| RECAP 标签 | 对应我们的类别 |
|-----------|-------------|
| 争辩-挑战 | A1 Challenging |
| 争辩-贬低 | A2 Discounting |
| 否认-责怪 | B1 Blaming |
| 否认-不认同 | B2 Disagreeing |
| 否认-找借口 | B3 Excusing |
| 否认-最小化 | B4 Minimizing |
| 否认-悲观 | B5 Pessimism |
| 否认-犹豫 | B6 Reluctance |
| 否认-不愿改变 | B7 Unwillingness |
| 回避-最小的回应 | C1 Minimum Talk |
| 回避-界限设定 | C2 Limit Setting |
| 忽视-岔开话题 | C3/D2（Topic Shift/Sidetracking）|
| 忽视-不关注 | D1 Inattention |
| 合作-未阻抗 | E2 Cooperative（暂定，需细分 E1/E3）|
