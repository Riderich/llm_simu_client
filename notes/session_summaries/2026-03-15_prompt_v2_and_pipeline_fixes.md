# Session Summary: Prompt 2.0 Evaluation & Pipeline Fixes
**Date**: 2026-03-15

## 1. 核心进展：Prompt 2.0 评估圆满成功
- **设计变更**：将所有 Inner Monologue 模板从第三人称“专家分析”视角转为**第一人称沉浸式视角**（`You ARE the patient`），并注入了由 `extract_persona.py` 提取的患者动态人设。
- **打分表现**：通过 `evaluate_inner_monologue.py` (LLM-as-a-Judge) 进行自动打分，平均分达到 **3.9~4.3 / 5.0**。
- **结论**：彻底解决了 AI “出戏”和“复读机”问题，独白自然度极大提升，可以定稿用于全量生成。

## 2. 关键 Bug 修复：打通全量生成逻辑
在测试过程中发现并解决了两个阻碍全量部署的问题：

### A. 数据标签“中英文脱落”修复
- **发现**：`AnnoMI-labeled.json` 中部分阻抗标签存为中文（如“否认”、“回避”），导致匹配逻辑失效，所有阻抗数据被错误分配到 `SIMPLE_AGREEMENT` 模板。
- **解决**：编写并执行了 `clean_annomi_labels.py`。
  - **一次性清洗**：将底层数据中的所有中文父类/子类标签（包括 `责备`, `贬低`, `设定限制` 等变体）翻译为标准英文词。
  - **效果**：实现底层数据与代码逻辑的完美解耦，彻底修复了错配问题。

### B. 极短对话引发的幻觉（Hallucination）拦截
- **发现**：针对只有 "Hi." 或 "Yeah." 的寒暄性转折，模型会强行生造复杂的内心动机。
- **解决**：在 `verify_psyfire_prompt.py` 中增加了**长度长度拦截器**。
  - **逻辑**：对词数 `< 4` 且不带阻抗属性的节点直接跳过生成，有效节省 API Token 并提高数据集纯度。
